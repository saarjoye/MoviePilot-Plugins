# DC助手多源版

作者：wYw

## 功能

- 支持动态新增、编辑、删除 DockerCopilot 源，适合 PVE 多 LXC 场景。
- 支持跨源容器更新通知、自动更新、进度汇报和镜像清理。
- 支持按源选择自动备份范围，留空则备份全部启用源。
- 容器选择值使用 `source_id::container_name`，避免多个 DC 中同名容器误更新。
- 详情页提供源状态、可更新容器、通知预览、选择摘要和失败源处理说明。

## 配置方式

在插件配置页按设计稿交互填写：

1. 在“基础开关”中配置启用状态、Cron 周期、通知开关、进度检查间隔和次数。
2. 在“DockerCopilot 源”区域点击“新增源”，每次新增或编辑一个 DC 源：
   - `源ID`：建议使用英文、数字、下划线，例如 `pve_lxc_01`。
   - `显示名称`：例如 `PVE-LXC-01`。
   - `服务地址`：DockerCopilot 服务地址，例如 `http://dc-lxc-01:12712`。
   - `secretKey`：DockerCopilot 环境变量中的密钥。
3. 保存后刷新插件配置页，容器选择区会按 `源名称 / 容器名` 加载可选项。
4. 在“容器选择”中分别配置更新通知、自动更新和备份源范围。

## 兼容说明

- 兼容 v1.0.0 的 `sources_text` JSON 配置，插件初始化时会自动迁移到动态源列表。
- 兼容 v1.0.1 的固定源配置，插件初始化时会自动迁移到动态 `sources` 列表。
- 兼容原单源插件的 `host` + `secretKey` 配置，迁移为默认源。
- 已保存的容器选择继续使用 `source_id::container_name`。

## 安全提醒

- `secretKey`、真实 DC 地址、LXC 主机信息属于敏感配置。
- 不要把真实配置截图或日志发到公开渠道。
- 插件通知、详情页和日志不会输出 `secretKey` 明文。
